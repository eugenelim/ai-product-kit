# `.claude/commands/_meta/`

Author-side scaffolding for the kit's slash commands. Nothing in this directory is a runnable slash command — these files are read by humans (and agents acting on behalf of humans) while *building* a new command, not at command-invocation time. Today the directory holds one file: `command-skeleton.md`, the literal body every Phase-4 template-fill command starts from per the convention in `docs/CONVENTIONS.md` §"Phase-4 Template-Fill Commands — `.claude/commands/draft-*.md` and siblings". To author a new in-scope command, copy `command-skeleton.md` to `.claude/commands/<verb>.md`, fill the `<placeholder>` markers, and walk the per-command spec checklist at `docs/specs/phase-4-command-convention/notes/per-command-spec-checklist.md`. Future command-build helpers (e.g., a `tools/new-command.sh` scaffolder) will land here as well if and when they're justified.
